# pnpm Monorepo Conversion Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Convert the auto-claude repo from a single root package to a pnpm workspace monorepo with `packages/daemon` and `packages/dashboard`, all within the `feature/dashboard` worktree before merging to main.

**Architecture:** Move daemon source (`src/`, `Dockerfile`, config files) into `packages/daemon/` and the existing `dashboard/` into `packages/dashboard/` using `git mv` to preserve history. Create a workspace root `package.json` + `pnpm-workspace.yaml`. Both Dockerfiles are rewritten to use the repo root as build context (required for access to the unified lockfile). The daemon's `WORKDIR` inside the container shifts to `/app/packages/daemon` so `process.cwd()` resolves `state/`, `prompts/`, and `fitness/` correctly.

**Tech Stack:** pnpm 10.32.1 workspaces, Docker multi-stage builds, `corepack`, `git mv`

**Working directory:** All commands run in `~/code/auto-claude/.worktrees/feature/dashboard` unless stated otherwise.

**Spec:** `docs/superpowers/specs/2026-03-19-pnpm-monorepo-design.md`

### Task 6: Rewrite daemon Dockerfile

**Files:**
- Rewrite: `packages/daemon/Dockerfile`

The old Dockerfile used `COPY package.json pnpm-lock.yaml ./` which assumes the build context is the package directory. The new Dockerfile uses the repo root as context (set in docker-compose), so `pnpm-lock.yaml` is accessible.

Key changes:
- Use `corepack enable` (reads pinned version from root `package.json` — no hardcoded pnpm version)
- Install phase at `WORKDIR /app` with all workspace manifests
- Switch to `WORKDIR /app/packages/daemon` before `CMD` so `process.cwd()` resolves `state/`, `prompts/`, `fitness/` correctly
- `CMD` (not `ENTRYPOINT`) so docker-compose `command:` can override it in dev

- [ ] **Step 1: Rewrite `packages/daemon/Dockerfile`**

Replace the entire file with:

```dockerfile
FROM node:22-slim

# corepack reads the pinned version from root package.json (pnpm@10.32.1)
RUN corepack enable
RUN apt-get update && apt-get install -y git curl && rm -rf /var/lib/apt/lists/*

# Install Claude CLI
RUN npm install -g @anthropic-ai/claude-code

# Install phase — build context is repo root so pnpm-lock.yaml is accessible
WORKDIR /app
COPY package.json pnpm-workspace.yaml pnpm-lock.yaml ./
COPY packages/daemon/package.json ./packages/daemon/package.json
COPY packages/dashboard/package.json ./packages/dashboard/package.json
RUN pnpm install --frozen-lockfile

# Copy daemon source only
COPY packages/daemon/src/ ./packages/daemon/src/
COPY packages/daemon/tsconfig.json ./packages/daemon/

# Run from the daemon package dir: process.cwd() resolves state/, prompts/, etc.
WORKDIR /app/packages/daemon
RUN mkdir -p state/runs

HEALTHCHECK --interval=30s --timeout=5s --start-period=10s \
  CMD curl -f http://127.0.0.1:3847/health || exit 1

EXPOSE 3847

# CMD (not ENTRYPOINT) so docker-compose command: can override in dev
CMD ["pnpm", "start", "--", "-c", "/app/packages/daemon/auto-claude.config.json"]
```

### Task 7: Rewrite dashboard Dockerfile

**Files:**
- Rewrite: `packages/dashboard/Dockerfile`

Key changes:
- Use `corepack enable` instead of `npm install -g pnpm`
- `deps` stage: copy all workspace manifests + unified lockfile from repo root
- `builder` stage: copy workspace root files so pnpm can resolve hoisted `node_modules` during `next build`
- `runner` stage: `WORKDIR /app/packages/dashboard` so `server.js` is found correctly

- [ ] **Step 1: Rewrite `packages/dashboard/Dockerfile`**

Replace the entire file with:

```dockerfile
FROM node:22-alpine AS base
# corepack reads the pinned version from root package.json (pnpm@10.32.1)
RUN corepack enable

FROM base AS deps
WORKDIR /app
COPY package.json pnpm-workspace.yaml pnpm-lock.yaml ./
COPY packages/daemon/package.json ./packages/daemon/package.json
COPY packages/dashboard/package.json ./packages/dashboard/package.json
RUN pnpm install --frozen-lockfile

FROM base AS builder
WORKDIR /app
# Workspace root files required: pnpm must see workspace context to resolve
# node_modules from /app/node_modules during next build
COPY package.json pnpm-workspace.yaml ./
COPY packages/daemon/package.json ./packages/daemon/package.json
COPY --from=deps /app/node_modules ./node_modules
COPY packages/dashboard/ ./packages/dashboard/
WORKDIR /app/packages/dashboard
RUN pnpm build

FROM base AS runner
WORKDIR /app/packages/dashboard
ENV NODE_ENV=production
COPY --from=builder /app/packages/dashboard/.next/standalone ./
COPY --from=builder /app/packages/dashboard/.next/static ./.next/static
COPY --from=builder /app/packages/dashboard/public ./public
EXPOSE 3000
CMD ["node", "server.js"]
```

**Implementation note:** After a real `docker build`, verify that `server.js` is at `/app/packages/dashboard/server.js` in the runner stage. In some pnpm monorepo configurations, Next.js standalone output nests the server under the workspace root path. If `server.js` is not found, update the `CMD` to the correct path based on the actual build output.

Expected first line: `FROM node:22-alpine AS base`
